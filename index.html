Google Hacking

VSB-TU Ostrava - 2023/24

What is Google Hacking?

Google Hacking, or Google Dorking, is a hacker technique to find information about vulnerable websites.

Google Hacking is not about hacking Google. It consists of using specific search commands with special search parameters and operators in Google to uncover information that is not usually found in regular Google Searches.

Basic Operators

The basis of Google Hacking is using the Google Search operators to find more specific information. However, there are a lot of operators, so we'll start with the more basic ones:
  • OR or | : Finds results related to one or two search terms. Try it: book OR computer.
  • " " : Finds results that include that exact phrase. Try it: "restaurant in ostrava"
  • in : Lets you convert an unit in another. Try it: 15 eur in czk
  • * : It is used as an wild card and fills the missing word or phrase. Try it: best * in Ostrava
  • - : Exclude a particular term or phrase from the search. Try it: web security -jobs
  • ... and more.

Advanced Operators

Now that we know some basic operators, we'll look at more advanced operators. This type of operators will be handy for the actual Google hacking. Some advanced operators are:
  • cache: - Exclude a particular term or phrase from the search. Try it: allintext:cyber threat
  • site: - Search in a specific Website. Try it: site:vsb.cz Computer Science
  • filetype: - Finds specific file types. Try it: "filetype:pdf web security"
  • intitle: - Finds pages with specific titles. Try it: intitle:”data privacy”
  • link: - Finds pages that link to a specific URL. Try it: link:https://www.vsb.cz/en/study/
  • intext: - Searches a specific text on Webpages. Try it: intext:computers
  • ... and more.

Examples : Log files

Many types of logs, like error logs or access logs, can be often discovered in the public space of websites. These logs can help hackers get information about the technologies used in that application and other types of critical data.
To find this we have to use two operatores, allintext and filetype, for example:
filetype:log allintext:username

Examples : Vulnerable Web Servers

We can find vulnerable web servers searching, for example, for websites that allow appending "/proc/self/cwd/" to the URL.
To find this we have to use the inurl operator for example:
inurl:/proc/self/cwd

Examples : Open FTP Servers

Google does not only index HTTP servers but also FTP servers which often hold large amounts of files and can contain interesting things.
To explore open FTP Servers we have to use two operatores, intitle and inurl, for example:
intitle:"index of" inurl:ftp

Examples : ENV Files

.env files are used by most web development frameworks to declare variables and configurations for local and online dev environments. The recommended practice it to move this files to a place that is not publicly accessible. However, there are a lot of cases that this doesn't happend and we can find a critical information in them.
To find this we have to use the filetype operator for example:
DB_USERNAME filetype:env

Examples : Live Cameras

Due to bad configuration, private live cameras can be watched not only by the owner but also by any person online. This happens because the IP of this live cameras' web pages is not restricted.
To find these live camera web pages we have to use the intitle operator, for example:
intitle:"webcamXP 5"

Further Atacks : DB Deletion

After finding all the settings and credentials from a database in a vulnerable ENV file is really easy to do anything to the database content. That's why is so dangerous to leave this kind of iformation so exposed. One example can be deleting the whole database with a command like this:

DROP DATABASE databasename;

Further Atacks : Brute Force FTP Credentials

After finding an open FTP server we can use tools like Hydra or Ncrack to find the credentials by Brute-Force. In this example, we will use Metasploitable-2 to simulate an open ftp server and Hydra to get its credentials. After finding an open ftp we only need to do some simple steps to get the credentials:
  • 1. Have two files with lists of common usernames and common passwords.
  • 2. Use this code on hydra where -L is used to set the username list and -P is used for the password list:

    hydra -L usernames.txt -P passwords.txt ftp://(serverIpAddress)

Preventions : Robots.txt

The robots.txt file is used to manage crawler traffic in a specific website. It contains a list of "rules" about who are allowed and who are not allowed to look in a website's directory structure. This means that the robots.txt file can give similar information to "Sitemaps". Here are a few keywords:
  • User-agent - Specify the type of crawler that can index the website.
  • Allow - Specify the files or directories that the crawler can index.
  • Disallow - Specify the files or directories that the crawler can't index.
  • Sitemap - Reference to where the Sitemap is located.

The robots.txt file should not be used as means to hide a webpage from Google. To do it we should use the noindex rule.

Preventions : GHDB

The Google Hacking Database(GHDB) is not a standand database, but rather a repository of operators, queries and search methods designed to locate specific categories of information and potential security flaws. The existence of this kind of repository helps developers and ethical hackers gain a deeper understanding of potential security risks in their websites.

Conlusions

To conclude, this project has highlighted the potential risks associated with the improper use of Google Hacking as they can be used for malicious activities such as data breaches, information gathering and unauthorized access. It is important for developers and organizations to be aware of the vulnerabilities exposed through this type of searches and take measures to safeguard their online presence.
